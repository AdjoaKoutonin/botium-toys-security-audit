# Botium Toys Security Audit & Risk Assessment

# Description

Conducted a cybersecurity risk assessment and controls audit for Botium Toys using the NIST Cybersecurity Framework (CSF). Evaluated existing security controls, identified compliance gaps related to PCI DSS, GDPR, and SOC standards, and provided recommendations to improve the organization's security posture.

## Objectives

* Assess organizational assets and security controls.
* Identify security risks and vulnerabilities.
* Evaluate compliance with PCI DSS, GDPR, and SOC standards.
* Determine gaps in security policies and procedures.
* Provide recommendations to reduce risk and improve security.

## Scope

The audit covered Botium Toys' entire security program, including:

* Employee devices and equipment
* Internal network infrastructure
* Data storage and retention systems
* E-commerce and inventory management systems
* Legacy systems requiring maintenance
* Physical security controls

## Risk Assessment Findings

The organization received a risk score of 8/10 due to several control and compliance deficiencies, including:

* Lack of least privilege access controls
* Absence of separation of duties
* No intrusion detection system (IDS)
* No disaster recovery plan
* No backups of critical data
* Lack of encryption for sensitive customer payment information
* Weak password management practices
* Inadequate asset inventory and classification processes

## Security Controls Evaluated

### Existing Controls

* Firewall
* Antivirus software
* Legacy system monitoring
* Physical locks
* CCTV surveillance
* Fire detection and prevention systems

### Missing Controls

* Least privilege
* Separation of duties
* Intrusion Detection System (IDS)
* Data encryption
* Password management system
* Disaster recovery planning
* Backup solutions

## Compliance Assessment

### PCI DSS

Identified compliance gaps involving:

* Access control to cardholder data
* Data encryption
* Secure storage and transmission of payment information
* Password management practices

### GDPR

Evaluated requirements related to:

* Data privacy and protection
* Data classification and inventory
* Breach notification procedures

### SOC Controls

Reviewed:

* User access controls
* Data confidentiality
* Data integrity
* Data availability

## Recommendations

To improve security and compliance, the following actions were recommended:

1. Implement least-privilege and role-based access control.
2. Establish separation of duties for critical processes.
3. Deploy an intrusion detection system (IDS).
4. Encrypt sensitive customer and payment data.
5. Implement centralized password management.
6. Create disaster recovery and business continuity plans.
7. Establish regular backup procedures.
8. Improve asset inventory and data classification processes.

Successfully completed a comprehensive security audit, identified critical control gaps, evaluated compliance requirements, and developed actionable recommendations to improve the organization's security posture and reduce business risk.
